# Codex Project Notes

## Working style

- Make small, reviewable changes.
- Explain assumptions before changing architecture.
- Prefer scripts that can be rerun safely.
- Do not store secrets, tokens, passwords, private keys, or auth files in git.
- Keep project documentation clear enough for a human to recover context without archaeology.

## Project direction

This repository is the workspace for Valera Control Tower:
- robotics PoC
- local AI agents
- enterprise integration demo
- GitHub-based product workflow
