# Roadmap

## Phase 0 — Repository foundation

Status: in progress

Goals:

- create project structure
- add README, CODEX notes, product brief, roadmap
- configure Codex workflow
- create GitHub repository
- establish small-commit discipline

## Phase 1 — Architecture and simulation

Goals:

- define system architecture
- define task model
- define robot state model
- create simulated robot execution
- create CLI demo for task lifecycle

Expected output:

- architecture document
- Python simulation prototype
- example task JSON files
- simple logs and result reports

## Phase 2 — Robot integration layer

Goals:

- define hardware abstraction
- separate simulation from real robot control
- prepare interfaces for tracked base and LeRobot arm
- document safety boundaries

Expected output:

- robot adapter interfaces
- mock adapter
- hardware notes
- basic command validation

## Phase 3 — Agent workflow

Goals:

- use Codex / local agents to assist development
- define agent roles
- create repeatable issue-to-PR workflow
- document agent constraints

Expected output:

- agent playbook
- issue templates
- development workflow notes

## Phase 4 — Enterprise integration PoC

Goals:

- model business task creation
- model event-driven status updates
- create integration-style API or message flow
- connect robotics execution with enterprise process language

Expected output:

- integration architecture
- API/event schemas
- demo process
- portfolio-ready explanation

## Phase 5 — Hardware demo

Goals:

- execute a safe real-world task
- record a short demo
- document what works and what is simulated
- prepare portfolio page

Expected output:

- demo video
- final README
- architecture diagram
- interview talking points
